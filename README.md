# Morning Brief Dashboard

A hosted dashboard for viewing daily morning briefings generated by Claude's scheduled tasks in Cowork.

## Quick Start — GitHub Pages (Free)

### 1. Create a GitHub repo

- Go to [github.com/new](https://github.com/new)
- Name it `morning-brief` (or whatever you like)
- Set it to **Private** (only you can see the code)
- Click **Create repository**

### 2. Push this folder to the repo

Open Terminal on your Mac and run:

```bash
cd ~/path/to/morning-brief-dashboard
git init
git add .
git commit -m "Initial dashboard setup"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/morning-brief.git
git push -u origin main
```

### 3. Enable GitHub Pages

- Go to your repo on GitHub → **Settings** → **Pages**
- Under "Source", select **Deploy from a branch**
- Branch: `main`, folder: `/ (root)`
- Click **Save**

Your dashboard will be live at: `https://YOUR_USERNAME.github.io/morning-brief/`

### 4. Update your Cowork scheduled task

Add this to the end of your scheduled task prompt so it auto-publishes each briefing:

> After generating the morning briefing HTML file, run the publish-brief.sh script
> in my morning-brief-dashboard repo to copy the file, update manifest.json, and
> push to GitHub. The repo is at ~/morning-brief-dashboard (or wherever you cloned it).

Or manually publish any briefing:

```bash
cd ~/morning-brief-dashboard
./publish-brief.sh ~/claude/Morning-Brief-2026-04-14.html
```

## Project Structure

```
morning-brief-dashboard/
├── index.html          ← Dashboard UI (loads in browser)
├── manifest.json       ← Registry of available briefings
├── publish-brief.sh    ← Script to add a new briefing + push
├── briefs/
│   ├── 2026-04-13.html ← Daily briefing files
│   └── ...
└── README.md
```

## How It Works

- `index.html` reads `manifest.json` on load to discover available briefings
- Briefings are displayed in a sidebar sorted newest-first
- The latest briefing auto-loads on page open
- The manifest auto-refreshes every 5 minutes to pick up new briefings
- `publish-brief.sh` handles copying, manifest updates, and git push

## Notes

- **Private repo**: GitHub Pages on private repos requires a paid GitHub plan ($4/mo) or GitHub Pro. If you want free hosting with a private repo, use Netlify instead (see below).
- **Custom domain**: You can add a custom domain in GitHub Pages settings if you want something cleaner than the github.io URL.
- **Auth**: For team access later, consider Cloudflare Access (free for up to 50 users) in front of the site.

## Alternative: Netlify (Also Free)

If you'd rather not use GitHub Pages:

1. Go to [app.netlify.com](https://app.netlify.com)
2. Drag and drop this entire folder onto the deploy area
3. Your site is live instantly

For automated deploys, install the Netlify CLI and add this to publish-brief.sh:
```bash
netlify deploy --prod --dir=.
```
